Unvalidated redirects and forwards

An attacker may use unvalidated redirects to forward unsuspecting users to malicious web sites. The attacker may then launch a phishing scam to steal user's credentials or other sensitive information or install malware on user's computer. They can also be used to bypass access checks and allow the attacker to gain access to priviledged functions.

Steps to identify

  • Click the following link click this link to get redirected
  • => A request is sent to the project site and the user's browser is then forwarded to an external web site.

Steps to fix the application

The application should not allow redirects or forwards to any url. Instead the url's should be checked against a whitelist so that only valid redirects are executed.
