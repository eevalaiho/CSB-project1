Broken Authentication and Session Management

Broken authentication and session management flaw occurs when an applications' functionalities related to authentication and session management are implemented improperly. This allows attackers to gain access to sensitive content such as passwords or session keys.

This flawed implementation allows an attacker to log in to the system without providing a password. This is due to a flaw in error handling management in the web application.

In order to reproduce the flaw you need to use a for example Burp suite to intercept and modify the web request.

Steps to execute

To log in using valid credentials:

  • enter larry in the username field and istheking in the password field and click Login.
  • => a welcome page is displayed.

To log in using without credentials:

  • enter larry in the username field and password in the password field and click Login.
  • => an error message is displayed.

To log in without valid credentials:

  • start and configure burp suite to intercept project site's traffic (see Securing Software, part 4 page for reference)
  • setup Burp Suite to intercept (Proxy -> Intercept -> Intercept is on/off)
  • modify the request so that you remove the password parameter
  • forward to original address.
  • => a welcome page is displayed.
